## 现在怎么开始

如果是从这个仓库重新进入学习，按这个顺序看：

1. 先看 `docs/roadmap-2026-2027.md`
2. 再看 `docs/week-01-python-basics.md`
3. 然后看 `learning_log.md`
4. 最后进入 `scripts/` 继续当天练习

如果只是想直接跑一个练习，可以从这里开始：

```powershell
python .\scripts\python_basics_day0.py
```

如果要看已经完成的两个小工具，可以运行：

```powershell
python .\scripts\site_probe.py https://example.com
python .\scripts\cert_days_left.py example.com
```

## 学习规则

- 先闭环，再扩展
- 所有代码都走 Git
- 每周必须有可运行结果
- AI 当陪练，不当替身
- 核心逻辑先自己写第一版
- 每周复盘：本周写了什么、卡在哪里、下周推进哪 1-2 件关键事

## 与 Codex 的协作规则

- 练习骨架、日志模板、复盘文档、README 结构，可以由 Codex 先搭好
- `python_basics_day0.py` 到 `hello_cli.py` 这类入门练习，Codex 可以先给练习文件，我负责修改、运行、解释和记录
- 从真实小工具开始，核心逻辑尽量由我先手敲
- 证书检测、数据库核心逻辑、认证逻辑、关键业务判断，都优先由我先写第一版
- Codex 可以做的事：拆步骤、解释概念、提示下一小步、检查报错、做 review、整理格式、补文档、补测试建议
- Codex 默认不要直接替我写完整核心答案，除非我明确说“先演示给我看”或者“这次你直接写一版给我学习”
- 如果我自己没写出来，Codex 先给最小提示，不直接给完整最终实现

判断是否算掌握，不看是不是一个字一个字手敲，而看这 4 点：

- 我能不能自己改
- 我能不能解释
- 我删掉后能不能重写核心部分
- 我遇到小 bug 能不能自己修回来
