# learning_plan
来自大一学生的学习计划：学习全栈偏后端与计算机四大件硬核进化计划，包含手敲核心逻辑与 AI 协同审计记录。
# 24周超级全栈（偏后端）与四大件硬核进化计划 🚀

本项目是一份将计算机四大件基础、全栈开发实战与 AI 协同深度结合的 24 周硬核自学进化大纲。本计划针对大一零基础、目前仅掌握 C++ 语言的实际情况进行了全面重构与量身定制。

我们的核心破局策略是：用目前唯一的武器 C++ 去正面硬刚底层的内存、指针和算法（发挥核心优势）；同时让 AI 扮演“跨语言翻译官”，带在一针一线中无痛降维通关 TypeScript (TS) 语法。

---

## 🎯 核心执行原则

1. 「禁区」法则 (Manual)：涉及数据结构、算法逻辑、底层操作系统与网络、核心安全及性能瓶颈的代码，绝对禁止使用任何 AI 自动补全（如 Copilot/Cursor 的 Tab 键）。这部分的核心目的是在脑海中强行建立底层的物理直觉。
2. AI 辅助生产 (Assisted)：涉及前端样板代码、UI 样式、简单测试用例、基础配置说明和常见 API 调用，则追求极致的生产速度。
3. AI 扮演“陪练”而非“替身”：永远不要让 AI 直接帮你写核心作业。正确做法是自己写完后，让 AI 基于计算机底层原理（如 CPU 缓存、内存模型）进行针对性的审计和优化。
4. 周日重构复盘：每周日选出本周由 AI 辅助编写的最长或最复杂的一段代码，将其彻底删掉，尝试根据自己理解的底层原理进行手敲复原。若无法复原，说明该知识点尚未学透。
5. 高时薪核心卖点：未来能接到单或者拿到好 Offer，是因为你懂“手敲”一栏里的底层机制（高并发不卡、分布式账本、数据安全），而不是因为你会拼凑简单的 UI 样式。

## 🗺️ 24周硬核筑基路线图

### 🛠️ 第一阶段：C++ 筑基与 TS 语法迁移（第 1 - 8 周）
重点：用 C++ 从零吃透数据结构与计算机组成原理，借助 C++ 的强类型直觉无痛降维通关 TS 语法。

### W1-W2：线性表与链表
1. 四大件核心：数据结构（线性表、链表）、组原（内存地址空间、野指针本质）
2. 🔥 手敲核心逻辑：C++ 实现双向链表，手敲完整的 insert、delete 逻辑，处理好 head 和 tail 的指针边界。
3. 🤖 AI 辅助与迁移：利用 AI 生成项目基础目录结构；让 AI 扮演翻译官，把 C++ 链表翻译成 TS 语法，死磕 TS 的变量声明与基础类型。
4. 🛡️ 零基础审计任务：审计 C++ 指针在 64 位系统下的物理内存占用，对比 TS 引用在 V8 引擎底层的分配差异。

### W3-W4：树与递归逻辑
1. 四大件核心：数据结构（树与递归）、组原（CPU 栈帧、函数调用代价）
2. 🔥 手敲核心逻辑：C++ 手写二叉搜索树（BST），脱离 AI 手敲树的节点插入以及前/中/后序的 DFS 遍历。
3. 🤖 AI 辅助与迁移：让 AI 生成复杂的二叉树 mock 数据；通过 AI 演示学习 TS 里的 class（类）和 interface（接口），对比 C++ 类的异同。
4. 🛡️ 零基础审计任务：审计递归深度对 CPU 栈帧的影响，弄清楚为什么递归过深会导致栈溢出（Stack Overflow）。

### W5-W6：哈希表与快速排序
1. 四大件核心：数据结构（哈希映射）、算法（经典快排）、组原（位运算与进制转换）
2. 🔥 手敲核心逻辑：C++ 手写快速排序算法；用 C++ 位运算实现一个状态管理器。
3. 🤖 AI 辅助与迁移：利用 AI 生成性能 Benchmark 脚本；在 AI 辅助下，亲手用 TS（数组+拉链法）实现一个基础哈希表，并熟悉 TS 内置 Map/Set 的 API。
4. 🛡️ 零基础审计任务：审计哈希冲突发生时，拉链法导致的链表过长对 CPU 缓存局部性（L1/L2 Cache）命中率的致命打击。

### W7-W8：阶段复盘与全栈初探
1. 四大件核心：算法精通与系统内存打通
2. 🔥 手敲核心逻辑：用 C++ 独立手写一个简易的哈希加密算法逻辑（如对字符串进行特定位运算异或加密）。
3. 🤖 AI 辅助与迁移：编写前端输入框；利用 AI 快速编写一个极其简单的网页界面（HTML/CSS），把你 C++ 的加密逻辑用 TS 移植并跑在浏览器里。
4. 🛡️ 零基础审计任务：审计加密计算过程中，系统对变量整型溢出（Overflow）的底层处理机制。

---

### 🌐 第二阶段：操作系统、网络与 Node.js 后端跨界（第 9 - 16 周）
重点：全面转入系统级开发。利用 C++ 沉淀的内存直觉，跨界死磕操作系统和网络协议，全面拥抱 Node.js（TS 后端）。

### W9-W10：网络协议与 epoll 模型
1. 四大件核心：计网（TCP/UDP、HTTP/3 演进）、OS（I/O 多路复用基石）
2. 🔥 手敲核心逻辑：手写 Node.js 原生 Socket 连接进行数据传输；手写一个 HTTP 请求拦截器。
3. 🤖 AI 辅助与生产：让 AI 全自动帮你生成前端 CSS/Tailwind 样式，高效率产出标准的登录界面。
4. 🛡️ 系统级审计任务：开启 Wireshark 抓包，深度审计 TCP 三次握手状态机变换。

### W11-W12：多线程并发锁与全双工通信
1. 四大件核心：OS（进程/线程、多线程并发锁）、计网（WebSocket 协议）
2. 🔥 手敲核心逻辑：手写多线程并发下的生产者-消费者模型（控制互斥）；手写 WebSocket 心跳检测机制。
3. 🤖 AI 辅助与生产：让 AI 生产实时聊天室的列表滚动逻辑，以及聊天的消息本地持久化代码。
4. 🛡️ 系统级审计任务：深度审计 AI 写的并发代码中是否存在导致死锁（Deadlock）的风险。

### W13-W14：虚拟内存与安全防御
1. 四大件核心：OS（虚拟内存、分页存储管理）、计网（网络安全防护）
2. 🔥 手敲核心逻辑：手写缓存置换算法（LRU）；手写后端的 SQL 注入防护和核心字符过滤逻辑。
3. 🤖 AI 辅助与生产：让 AI 自动化编写数据库表迁移（Migration）脚本和表结构声明。
4. 🛡️ 系统级审计任务：审计数据库索引（B+ Tree）在磁盘 I/O 上的物理表现。

### W15-W16：后端鉴权与阶段复盘
1. 四大件核心：全栈进阶体系打通
2. 🔥 手敲核心逻辑：脱离 AI，全手写后端 API 的 JWT（Json Web Token）鉴权与校验逻辑。
3. 🤖 AI 辅助与生产：让 AI 根据你的数据库结构，全自动生成全套后台 CRUD（增删改查）页面模板。
4. 🛡️ 系统级审计任务：审计 AI 生成的代码在异常处理（Try-Catch）和错误边界上是否完备安全。

---

### 🚀 第三阶段：高级算法、分布式与全栈商业化（第 17 - 24 周）
重点：冲刺行业高薪核心技术。死磕大厂与商业项目看重的：AI 向量检索、Rust 内存安全与 Web3 智能合约。

### W17-W18：高阶算法与 RAG 架构
1. 四大件核心：高级算法（向量索引、RAG）、组原（GPU 并行计算原理）
2. 🔥 手敲核心逻辑：手写文本余弦相似度计算逻辑；脱离第三方库手写一个基础的向量检索流动过程。
3. 🤖 AI 辅助与生产：生成连接云端向量数据库（Pinecone/Milvus）的配置和基础连接脚手架。
4. 🛡️ 商业级审计任务：审计 AI 对大规模文本进行切片（Chunking）时的逻辑是否导致信息丢失。

### W19-W20：Rust 内存安全与分布式共识
1. 四大件核心：OS（Rust 内存所有权模型）、算法（Raft 分布式共识机制）
2. 🔥 手敲核心逻辑：手写 Rust 内存所有权转换示例（生命周期）；实现一个简易的 Raft 选举领袖核心逻辑。
3. 🤖 AI 辅助与生产：让 AI 生成智能助手的对话前端 UI 界面及前端全局状态 management 逻辑。
4. 🛡️ 商业级审计任务：深度审计 Rust 编译器报错，对比理解其如何在编译期消灭 C++ 里的内存泄漏和悬空指针。

### W21-W22：Web3 智能合约与密码学
1. 四大件核心：计网（P2P协议、ECC加密）、Web3（智能合约）
2. 🔥 手敲核心逻辑：手写区块链智能合约的核心交易与资产转移逻辑（使用 Solidity 或 Rust）。
3. 🤖 AI 辅助与生产：让 AI 生成前端网页与区块链钱包交互的 Web3.js 冗长样板代码。
4. 🛡️ 商业级审计任务：深度审计合约的 Gas 消耗与防范黑客重入攻击（Reentrancy）的安全漏洞。

### W23-W24：终极毕设与全链路高并发压测
1. 四大件核心：毕设架构与商业化变现
2. 🔥 手敲核心逻辑：独立架构和手写整个系统的全链路安全防护网、数据加密流和防刷限流逻辑。
3. 🤖 AI 辅助与生产：让 AI 全自动帮你生成项目的完整 API 文档（Swagger 手册）和 Docker 自动化部署脚本。
4. 🛡️ 商业级审计任务：全链路性能审计，定位并审计从服务器网卡接收到数据包再到 CPU 运算的每一个微秒级瓶颈。
